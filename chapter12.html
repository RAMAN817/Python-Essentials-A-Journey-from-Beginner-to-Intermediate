Chapter 12: OOP- Part One

Introduction

Object-Oriented Programming (OOP) is a programming paradigm that provides a means of structuring programs so that properties and behaviors are bundled into individual objects. In Python, OOP is a fundamental part of the language. This chapter introduces the basic concepts of OOP in Python, focusing on classes, objects, and reference variables.

Class, Object, and Reference Variables

Class

A class is a blueprint or template for creating objects. It defines the data members (attributes) and behaviors (methods) that the objects created from the class can have.

In other words, a class is a logical grouping of data and functions that act upon that data.

Object

An object is an instance of a class. It is a physical representation of the class in memory. Objects have their own state and can interact with other objects.

Reference Variable

A reference variable is used to refer to an object. By using reference variables, we can access the properties and methods of an object. Multiple reference variables can refer to the same object.

Defining a Class in Python

You can define a class in Python using the class keyword:


    class ClassName:
    '''Documentation string (optional)'''
    # Variables (attributes)
    # Methods (behaviors)

The documentation string (docstring) is optional and can be accessed using ClassName.__doc__ or help(ClassName).

Example


    class Student:
    '''This is a Student class.'''

    # Variables and methods go here

Variables in a Class

There are three types of variables in a class:

  1. Instance Variables: Variables that are unique to each object (instance).
  2. Static Variables: Class-level variables shared among all instances.
  3. Local Variables: Variables defined within methods and only accessible there.

Methods in a Class

Methods are functions defined inside a class that describe the behaviors of the objects.

  1. Instance Methods: Methods that act upon instance variables.
  2. Class Methods: Methods that act upon class variables and are bound to the class rather than the object instance.
  3. Static Methods: General utility methods that perform a task in isolation. They do not access or modify class state.

Example: Defining and Using a Class


    class Student:
    '''This is the Student class.'''

    def __init__(self):
        self.name = 'Roman'
        self.roll_no = 1010
        self.marks = 90

    def talk(self):
        print('Hello, my name is', self.name)
        print('My roll number is', self.roll_no)
        print('My marks are', self.marks)

# Creating an object (instance) of the Student class
s = Student()

# Accessing instance variables
print(s.name)      # Output: Roman
print(s.roll_no)   # Output: 1010
print(s.marks)     # Output: 90

# Calling an instance method
s.talk()

Output:


Hello, my name is Roman
My roll number is 1010
My marks are 90

The self Variable

Understanding self

The self parameter refers to the instance of the class. It is used to access variables that belong to the class. In Python, self is not a keyword; it's just a convention. You can use any name instead of self, but it's highly recommended to stick with self to maintain code readability and consistency.

Inside any instance method, the first parameter must be self (or any other name you choose), which refers to the object itself.

Proof of self Pointing to the Current Object


    class Test:
    def __init__(self):
        print('Constructor called')
        print('ID of self:', id(self))

# Creating an object
t = Test()
print('ID of t:', id(t))

Output:


Constructor called
ID of self: 140435671525584
ID of t: 140435671525584

The IDs of self and t are the same, proving that self refers to the current object.

Example with Parameters


    class Student:
    def __init__(self, name, roll_no, marks):
        self.name = name
        self.roll_no = roll_no
        self.marks = marks

    def talk(self):
        print('Hello, my name is', self.name)
        print('My roll number is', self.roll_no)
        print('My marks are', self.marks)

# Creating an object with parameters
s1 = Student('Sunny', 101, 90)
s1.talk()

Output:


Hello, my name is Sunny
My roll number is 101
My marks are 90

Customizing self (Not Recommended)

Although you can use any name instead of self, it's not recommended as it can make the code less readable.


    class Student:
    def __init__(this, name, roll_no, marks):
        this.name = name
        this.roll_no = roll_no
        this.marks = marks

    def talk(this):
        print('Hello, my name is', this.name)
        print('My roll number is', this.roll_no)
        print('My marks are', this.marks)

s1 = Student('Sunny', 101, 90)
s1.talk()

Note: Even though this code works, it's best to stick with self.

Constructors in Python

The __init__ Method

The __init__ method is a special method in Python classes, known as the constructor. It is automatically called when an object is created. The purpose of the constructor is to initialize the object's attributes.


    class Test:
    def __init__(self):
        print('Constructor executed')

t = Test()
# Output: Constructor executed

Purpose of Constructors

  • Initialize instance variables.
  • Set up initial state of the object.

Optional Constructors

If you do not define a constructor, Python provides a default constructor that does nothing.


    class Test:
    def m1(self):
        print('Method m1 executed')

t = Test()
t.m1()
# Output: Method m1 executed

Overloading Constructors

Python does not support constructor overloading; you cannot have multiple constructors in a class. If you define multiple __init__ methods, the last one will overwrite the previous ones.

However, you can provide default values for parameters or use variable-length arguments to simulate overloading.


    class Test:
    def __init__(self, a=None, b=None, c=None):
        self.a = a
        self.b = b
        self.c = c

t1 = Test()
t2 = Test(10)
t3 = Test(10, 20)
t4 = Test(10, 20, 30)

Difference Between Methods and Constructors

Constructors Methods
  • Special method named __init__.
  • Called automatically when an object is created.
  • Used to initialize instance variables.
  • Only one constructor per class (no overloading).
  • Regular functions defined within a class.
  • Called explicitly using the object reference.
  • Used to define behaviors of the object.
  • Can have multiple methods with different names.
  • Can be called multiple times.

Key Takeaways

  • A class is a blueprint for creating objects.
  • An object is an instance of a class.
  • Reference variables are used to refer to objects.
  • The self variable refers to the current instance of the class.
  • Constructors are used to initialize the object's state.
  • Methods define the behaviors of the objects.
  • Understanding the use of self and constructors is crucial in OOP.

Conclusion

This chapter introduced the basic concepts of Object-Oriented Programming in Python, focusing on classes, objects, reference variables, and constructors. In the next chapters, we will delve deeper into other OOP concepts like inheritance, polymorphism, encapsulation, and abstraction.
