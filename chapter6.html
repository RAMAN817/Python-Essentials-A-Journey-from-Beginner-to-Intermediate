Chapter 6: Tuples

Introduction to Tuples

A tuple in Python is similar to a list. It shares the following properties with lists:

  • Order is preserved: Elements have a defined order and can be accessed via indices.
  • Duplicates are allowed: A tuple can contain multiple instances of the same element.
  • Heterogeneous objects: Elements can be of different data types.
  • Indexing and slicing are applicable: Elements can be accessed and manipulated using indices and slicing.

The key difference between tuples and lists is that tuples are immutable. Once a tuple is created, its contents cannot be changed. Tuples are represented using parentheses (). Note that parentheses are optional; commas define the tuple.

a = 10, 20, 'roman'
print(type(a))  # Output: <class 'tuple'>

Use Cases

Lists are suitable when you need a collection of items that can change over time. Examples include:

  • YouTube comment sections
  • Twitter comments

Tuples are suitable when you need a collection of items that should not change. Examples include:

  • Allowed input values for a vending machine
  • Coordinates in a game

Creating Tuples

Single-Valued Tuple

To create a tuple with a single value, include a comma after the value. Parentheses are optional but recommended for clarity.

t = (10,)
print(type(t))  # Output: <class 'tuple'>

t = 10,
print(type(t))  # Output: <class 'tuple'>

t = (10)
print(type(t))  # Output: <class 'int'> (Not a tuple)

Empty Tuple

t = ()
print(type(t))  # Output: <class 'tuple'>

Multi-Valued Tuple

t = (10, 20, 30)
t = 10, 20, 30
t = (10, 20, 30,)

Using the tuple() Function

You can create a tuple from a sequence using the tuple() constructor.

l = [10, 20, 30, 40, 50, 60]
t = tuple(l)
print(t)  # Output: (10, 20, 30, 40, 50, 60)

With Dynamic Input

t = eval(input('Enter tuple of values: '))
print(t)

Accessing Elements of a Tuple

1. By Using Index

t = (10, 20, 30, 40, 50, 60)
print(t[0])    # Output: 10
print(t[-1])   # Output: 60
print(t[5])    # Output: 60
# print(t[100])  # Raises IndexError

2. By Using Slice Operators

t = (10, 20, 30, 40, 50, 60, 70, 80)
print(t[2:5])   # Output: (30, 40, 50)
print(t[::2])   # Output: (10, 30, 50, 70)

Mathematical Operators for Tuples

1. Concatenation Operator (+)

Tuples can be concatenated using the + operator. Both operands must be tuples.

t1 = (10, 20, 30)
t2 = (40, 50, 60)
t3 = t1 + t2
print(t3)  # Output: (10, 20, 30, 40, 50, 60)

# Invalid concatenations:
# t3 = t1 + 10        # TypeError
# t3 = t1 + [30, 30]  # TypeError

2. Repetition Operator (*)

You can repeat a tuple multiple times using the * operator.

t1 = (10, 20, 30)
t3 = t1 * 3
print(t3)  # Output: (10, 20, 30, 10, 20, 30, 10, 20, 30)

Equality Operators for Tuples

Using == and != operators:

  1. The contents must be the same.
  2. The order must be the same.
  3. The elements must be the same.
l1 = ('a', 'b', 'c')
l2 = ('A', 'B', 'C')
l3 = ('c', 'a', 'b')
l4 = ('a', 'b', 'c')

print(l1 == l2)  # Output: False
print(l1 == l3)  # Output: False
print(l1 == l4)  # Output: True

Relational Operators

Using <, >, <=, >= operators. Comparisons are made based on the content of the tuples, element by element.

l1 = (10, 20, 30, 40)
l2 = (50, 60)

print(l1 < l2)   # Output: True
print(l1 <= l2)  # Output: True
print(l1 > l2)   # Output: False
print(l1 >= l2)  # Output: False

Example with strings:

l1 = ('Roman', 'son')
l2 = ('Raman', 'son')

print(l1 > l2)  # Output: True (Because 'o' > 'a' in 'Roman' vs 'Raman')

Membership Operators

Using in and not in operators:

l1 = (10, 20, 30, 40, 80)
print(50 in l1)        # Output: False
print(50 not in l1)    # Output: True

Important Methods and Functions

1. len() Function

Returns the number of elements in the tuple.

t = (10, 20, 30, 40)
print(len(t))  # Output: 4

2. count() Method

Returns the number of times a specified value occurs in a tuple.

t = (10, 20, 10, 30, 10)
print(t.count(10))  # Output: 3

3. index() Method

Returns the index of the first occurrence of a specified value.

t = (10, 20, 30, 40)
print(t.index(30))  # Output: 2
# print(t.index(50))  # Raises ValueError

Reversing Elements of a Tuple

Tuples do not have a reverse() method since they are immutable. However, you can reverse a tuple using the reversed() function and then converting the result back to a tuple.

t = (10, 20, 30, 40)
# t.reverse()  # AttributeError

r = reversed(t)
t1 = tuple(r)
print(t)   # Output: (10, 20, 30, 40)
print(t1)  # Output: (40, 30, 20, 10)

Sorting a Tuple

Since tuples are immutable, you cannot sort them in place. However, you can use the sorted() function to get a sorted list and then convert it back to a tuple.

t = (20, 5, 10, 0, 15)
s = sorted(t)
t1 = tuple(s)
print(t1)  # Output: (0, 5, 10, 15, 20)

To sort in descending order:

s = sorted(t, reverse=True)
t1 = tuple(s)
print(t1)  # Output: (20, 15, 10, 5, 0)

max() and min() Functions for Tuples

You can use the built-in max() and min() functions to find the maximum and minimum values in a tuple.

t = (20, 5, 10, 0, 15)
print(max(t))  # Output: 20
print(min(t))  # Output: 0

Tuple Packing and Unpacking

Packing

Assigning multiple values to a single tuple variable.

a = 10
b = 20
c = 30
d = 40
t = a, b, c, d
print(t)  # Output: (10, 20, 30, 40)

# For lists:
l = [a, b, c, d]
print(l)  # Output: [10, 20, 30, 40]

Unpacking

Assigning elements of a tuple to multiple variables.

t = (10, 20, 30, 40)
a, b, c, d = t
print(a, b, c, d)  # Output: 10 20 30 40

# Incorrect unpacking:
# a, b, c = t       # ValueError
# a, b, c, d, e = t # ValueError

# Using the asterisk (*) operator:
a, *b = t
print(a)  # Output: 10
print(b)  # Output: [20, 30, 40]

Tuple Comprehension

Tuple comprehension is not directly supported in Python. If you use comprehension syntax with parentheses, you get a generator object instead.

t = (x for x in range(2, 6))
print(type(t))  # Output: <class 'generator'>

# To create a tuple, you can use:
t = tuple(x for x in range(2, 6))
print(t)  # Output: (2, 3, 4, 5)

Additional Notes

Searching Methods

In sets and dictionaries, order is not important. They use hashing for efficient search operations. Here's a brief comparison of different searching methods:

  • Linear Search: Checks each element one by one. Time complexity is O(n).
  • Binary Search: Works on sorted sequences. Divides the search interval in half each time. Time complexity is O(log n).
  • Hashing: Uses a hash function to map keys to values. Provides very efficient search, insertion, and deletion. Time complexity is O(1) on average.

Hashability

Lists are unhashable because they are mutable, whereas tuples are hashable because they are immutable.

import collections

l = [10, 20, 30]
t = (10, 20, 30)

print(isinstance(l, collections.Hashable))  # Output: False
print(isinstance(t, collections.Hashable))  # Output: True

# Getting the hash value of a tuple
print(hash(t))  # Output: (An integer hash value)

Adding Tuples and Lists to a Set

You cannot add a list to a set because lists are unhashable. Tuples can be added to a set.

l = [10, 20, 30]
t = (10, 20, 30)
s = {10, 20}

# s.add(l)  # TypeError: unhashable type: 'list'
s.add(t)
print(s)  # Output: {10, 20, (10, 20, 30)}

Example Program

Write a program to take a tuple of numbers from the keyboard and print its sum and average.

# Taking input
t = eval(input('Enter tuple of numbers: '))

# Calculating sum and average
total = 0
for x in t:
    total += x

print('The sum:', total)
print('The average:', total / len(t))

Comparison Between Lists and Tuples

List Tuple
  1. Lists are groups of comma-separated values within square brackets. Square brackets are mandatory.
    Example: i = [10, 20, 30, 40]
  2. List objects are mutable; you can perform changes after creation.
    Example: i[1] = 70
  3. If the content is not fixed and keeps changing, use a list.
  4. List objects cannot be used as keys for dictionaries because they are unhashable and mutable.
  1. Tuples are groups of comma-separated values within parentheses. Parentheses are optional.
    Example: t = (10, 20, 30, 40) or t = 10, 20, 30, 40
  2. Tuple objects are immutable; you cannot change their content after creation.
    Example: t[1] = 70 raises TypeError
  3. If the content is fixed and never changes, use a tuple.
  4. Tuple objects can be used as keys for dictionaries because they are hashable and immutable.
