Chapter 13: OOP- Part Two

Introduction

In the previous chapter, we introduced the basics of Object-Oriented Programming (OOP) in Python, focusing on classes, objects, and constructors. In this chapter, we delve deeper into OOP by exploring different types of variables and methods within classes.

Types of Variables in Python OOP

Variables in a Python class can be classified into three main types:

  1. Instance Variables
  2. Class Variables (Static Variables)
  3. Local Variables

1. Instance Variables

An instance variable is a variable that is unique to each object (instance) of a class. If the value of a variable differs from one object to another, it is an instance variable. These variables are typically defined within methods using the self keyword.

Characteristics of Instance Variables

  • Declared within the constructor (__init__) or other instance methods using self.
  • Each object has its own copy of the instance variables.
  • Used to maintain state information specific to an object.

Example

class Student:
    def __init__(self, name, roll_no):
        self.name = name        # Instance variable
        self.roll_no = roll_no  # Instance variable

# Creating objects
student1 = Student('Alice', 101)
student2 = Student('Bob', 102)

# Accessing instance variables
print(student1.name)    # Output: Alice
print(student2.name)    # Output: Bob

2. Class Variables (Static Variables)

A class variable, also known as a static variable, is shared among all instances of a class. If the value of a variable is the same for all objects, it is a class variable. These variables are defined within the class but outside any methods.

Characteristics of Class Variables

  • Declared inside the class but outside of any methods.
  • Shared by all instances of the class.
  • Accessed using the class name or an instance.

Example


    class Student:
    school_name = 'DurgaSoft'  # Class variable

    def __init__(self, name, roll_no):
        self.name = name
        self.roll_no = roll_no

# Accessing class variable
print(Student.school_name)  # Output: DurgaSoft

student1 = Student('Alice', 101)
print(student1.school_name)  # Output: DurgaSoft

3. Local Variables

Local variables are variables declared inside a method and are only accessible within that method. They are used to store temporary data needed during method execution.

Example


    class Calculator:
    def add(self, a, b):
        result = a + b  # Local variable
        print('Sum:', result)

calc = Calculator()
calc.add(10, 20)  # Output: Sum: 30

Types of Methods in Python OOP

Methods in a Python class can be categorized into three types:

  1. Instance Methods
  2. Class Methods
  3. Static Methods

1. Instance Methods

An instance method is a method that operates on instance variables of an object. These methods require an object of the class to be created before they can be called. The first parameter of an instance method is always self.

Example


    class Student:
    def __init__(self, name, roll_no):
        self.name = name
        self.roll_no = roll_no

    def get_student_info(self):
        print('Student Name:', self.name)
        print('Student Roll No:', self.roll_no)

student = Student('Alice', 101)
student.get_student_info()

Output


Student Name: Alice
Student Roll No: 101

2. Class Methods

A class method is a method that operates on the class variables (static variables). It doesn't require an instance of the class to be created. Class methods are declared using the @classmethod decorator, and the first parameter is cls, which refers to the class itself.

Example


    class Student:
    school_name = 'Lambton'  # Class variable

    @classmethod
    def get_school_name(cls):
        print('School Name:', cls.school_name)

Student.get_school_name()

Output


School Name: Lambton

Explanation

  • The @classmethod decorator is used to declare a method as a class method.
  • The cls parameter refers to the class, not the instance.
  • Class methods can access and modify class variables.

Proof of cls Referring to the Class Object


    class Test:
    @classmethod
    def display_class_id(cls):
        print('ID of cls:', id(cls))

print('ID of Test:', id(Test))
Test.display_class_id()

Output


ID of Test: 140255600765136
ID of cls: 140255600765136

This shows that cls refers to the class object itself.

3. Static Methods

A static method is a general utility method that doesn't access any class or instance variables. Static methods are declared using the @staticmethod decorator.

Example


    class MathUtils:
    @staticmethod
    def add(a, b):
        return a + b

result = MathUtils.add(10, 20)
print('Sum:', result)  # Output: Sum: 30

Explanation

  • The @staticmethod decorator is used to declare a static method.
  • Static methods do not have access to self or cls.
  • They behave like regular functions but belong to the class's namespace.

Declaring Instance Variables

Instance variables can be declared in various places within a class:

1. Inside the Constructor Using self


    class Test:
    def __init__(self):
        self.a = 10
        self.b = 20

t = Test()
print(t.__dict__)  # Output: {'a': 10, 'b': 20}

The __dict__ attribute contains a dictionary of the instance's variables.

2. Inside an Instance Method Using self


    class Test:
    def __init__(self):
        self.a = 10
        self.b = 20

    def set_c(self):
        self.c = 30

t = Test()
t.set_c()
print(t.__dict__)  # Output: {'a': 10, 'b': 20, 'c': 30}

3. Outside the Class Using the Object Reference


    class Test:
    def __init__(self):
        self.a = 10
        self.b = 20

t = Test()
t.c = 30  # Adding an instance variable outside the class
print(t.__dict__)  # Output: {'a': 10, 'b': 20, 'c': 30}

Note

The number of instance variables can vary between objects.


t1 = Test()
t1.d = 40
print(t1.__dict__)  # Output: {'a': 10, 'b': 20, 'd': 40}

t2 = Test()
print(t2.__dict__)  # Output: {'a': 10, 'b': 20}

Accessing, Updating, and Deleting Instance Variables

Accessing Instance Variables


print(t.a)  # Accessing instance variable 'a'

Updating Instance Variables


t.a = 100  # Updating instance variable 'a'
print(t.a)   # Output: 100

Deleting Instance Variables

Within the Class


    class Test:
    def __init__(self):
        self.a = 10
        self.b = 20

    def delete_b(self):
        del self.b

t = Test()
t.delete_b()
print(t.__dict__)  # Output: {'a': 10}

Outside the Class


del t.a  # Deleting instance variable 'a'
print(t.__dict__)  # Output: {}

Declaring Static Variables

Static variables can be declared in several ways:

1. Inside the Class but Outside Any Methods


    class Test:
    a = 10  # Static variable

2. Inside the Constructor Using the Class Name


    class Test:
    def __init__(self):
        Test.b = 20  # Static variable

t = Test()
print(Test.b)  # Output: 20

3. Inside an Instance Method Using the Class Name


    class Test:
    def set_c(self):
        Test.c = 30  # Static variable

t = Test()
t.set_c()
print(Test.c)  # Output: 30

4. Inside a Class Method Using cls or Class Name


    class Test:
    @classmethod
    def set_d(cls):
        cls.d = 40     # Using cls
        Test.e = 50    # Using class name

Test.set_d()
print(Test.d)  # Output: 40
print(Test.e)  # Output: 50

5. Inside a Static Method Using the Class Name


    class Test:
    @staticmethod
    def set_f():
        Test.f = 60  # Static variable

Test.set_f()
print(Test.f)  # Output: 60

6. Outside the Class Using the Class Name


Test.g = 70
print(Test.g)  # Output: 70

Accessing Static Variables

Static variables can be accessed using the class name or an object reference. However, it is recommended to use the class name.

Inside the Class

Using self or Class Name in the Constructor


    class Test:
    a = 10

    def __init__(self):
        print(self.a)    # Access via self
        print(Test.a)    # Access via class name

Using self or Class Name in an Instance Method


    class Test:
    a = 10

    def display_a(self):
        print(self.a)    # Access via self
        print(Test.a)    # Access via class name

Using cls or Class Name in a Class Method


    class Test:
    a = 10

    @classmethod
    def display_a(cls):
        print(cls.a)     # Access via cls
        print(Test.a)    # Access via class name

Outside the Class


t = Test()
print(t.a)      # Access via object reference
print(Test.a)   # Access via class name (recommended)

Modifying Static Variables

Static variables can be modified anywhere using the class name or cls (inside class methods).

Example


    class Test:
    a = 10

    @classmethod
    def modify_a(cls):
        cls.a = 20  # Modifying static variable using cls

Test.modify_a()
print(Test.a)  # Output: 20

Test.a = 30    # Modifying static variable using class name
print(Test.a)  # Output: 30

Difference Between Instance and Static Variables

Instance Variables Static Variables
  • Object-level variables.
  • Separate copy for each object.
  • Defined using self.
  • Values can differ between objects.
  • Class-level variables.
  • Shared among all objects.
  • Defined using class name or within the class.
  • Same value for all objects unless modified.

Deleting Static Variables

Static variables can be deleted using the del statement and the class name.


    class Test:
    a = 10

del Test.a  # Deleting static variable 'a'

# Accessing Test.a now will raise an AttributeError
# print(Test.a)  # AttributeError: type object 'Test' has no attribute 'a'

Local Variables

Local variables are variables declared inside a method and are only accessible within that method. They are used for temporary computations or storage.

Example


    class Test:
    @staticmethod
    def average(numbers):
        total = sum(numbers)          # Local variable
        count = len(numbers)          # Local variable
        result = total / count        # Local variable
        print('The average is', result)

numbers = [10, 20, 30, 40, 50]
Test.average(numbers)

Output

The average is 30.0

In this example, total, count, and result are local variables used within the average method.

Conclusion

Understanding the different types of variables and methods in Python OOP is crucial for designing robust and maintainable classes. Instance variables and methods allow objects to maintain their own state and behavior, while static variables and methods provide shared data and functionality across all instances. Local variables help manage temporary data within methods.

By leveraging these concepts, you can create classes that are efficient, organized, and aligned with the principles of object-oriented programming.
