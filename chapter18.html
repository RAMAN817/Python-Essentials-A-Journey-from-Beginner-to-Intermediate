Chapter 18: Polymorphism in Python

Introduction

Polymorphism is a core concept in object-oriented programming (OOP) that refers to the ability of a single entity (such as a method, operator, or object) to take on multiple forms. The term originates from the Greek words poly (meaning "many") and morph (meaning "form"). In Python, polymorphism allows us to define methods in a child class that have the same name as the methods in the parent class. It also allows operators to have different meanings based on the context.

Examples of Polymorphism

Operator Overloading

Operator overloading is a form of polymorphism where the same operator behaves differently based on the operands. In Python, operators like + can be used with different data types and perform different operations.


# Adding integers
result = 10 + 20
print(result)  # Output: 30

# Concatenating strings
result = 'durga' + 'soft'
print(result)  # Output: durgasoft

In the above example, the + operator adds two integers when both operands are numbers and concatenates two strings when both operands are strings. This is an example of polymorphism in action.

Method Overriding

Method overriding allows a child class to provide a specific implementation of a method that is already defined in its parent class. This is another form of polymorphism.


class Parent:
    def marry(self):
        print('Subbalakshmi or Appalamma')

class Child(Parent):
    def marry(self):
        print('Katrina Kaif')

c = Child()
c.marry()

Output:


Katrina Kaif

In this example, the Child class overrides the marry method of the Parent class to provide its own implementation.

Operator Overloading in Python

Python allows us to change the default behavior of operators depending on the operands. This is known as operator overloading. You can overload operators by defining special methods in your class.

Example: Overloading the + Operator

Suppose we have a Book class, and we want to add two Book objects to get the total number of pages.

class Book:
    def __init__(self, pages):
        self.pages = pages

    def __add__(self, other):
        total_pages = self.pages + other.pages
        return total_pages

b1 = Book(100)
b2 = Book(200)
print(b1 + b2)  # Output: 300

b3 = Book(500)
print(b1 + b3)  # Output: 600

In this example, we overloaded the + operator by defining the __add__() method. When we use b1 + b2, it calls b1.__add__(b2), which returns the sum of pages.

The __str__() Method

The __str__() method is a special method in Python that returns a string representation of an object. It is called by the built-in str() function and by the print() function.

Example


class Student:
    def __init__(self, name, rollno, marks):
        self.name = name
        self.rollno = rollno
        self.marks = marks

    def __str__(self):
        return 'Name: {}, Roll No: {}, Marks: {}'.format(self.name, self.rollno, self.marks)

s1 = Student('Ram', 102, 239)
s2 = Student('Roman', 101, 348)

print(s1)
print(s2)

Output:


Name: Ram, Roll No: 102, Marks: 239
Name: Roman, Roll No: 101, Marks: 348

By overriding the __str__() method, we provide a meaningful string representation of the object when it is printed.

Example: Overloading + and __str__() in Book Class


class Book:
    def __init__(self, pages):
        self.pages = pages

    def __add__(self, other):
        return Book(self.pages + other.pages)

    def __mul__(self, other):
        return Book(self.pages * other.pages)

    def __str__(self):
        return 'Total number of pages: {}'.format(self.pages)

b1 = Book(100)
b2 = Book(200)
b3 = Book(500)

print(b1 + b2 + b3)  # Output: Total number of pages: 800
print(b1 + b2 * b3)  # Output: Total number of pages: 100100

In this example, we overloaded the __add__() and __mul__() methods to allow addition and multiplication of Book objects. We also overrode the __str__() method to provide a meaningful string when printing the object.

Method Overloading in Python

Method overloading refers to the ability to have multiple methods with the same name but different parameters. While some programming languages like Java support method overloading, Python does not support it in the same way.

Why Python Doesn't Support Method Overloading

In Python, you cannot have multiple methods with the same name in a class. If you try to define methods with the same name, the last method defined will overwrite the previous ones.

Example


class Test:
    def m1(self):
        print('No-argument method')

    def m1(self, x):
        print('One-argument method')

    def m1(self, x, y):
        print('Two-argument method')

t = Test()
t.m1()  # TypeError: m1() missing 2 required positional arguments: 'x' and 'y'

In the above code, the last method m1(self, x, y) overwrites the previous definitions of m1. Therefore, when we try to call t.m1(), it raises a TypeError because it expects two arguments.

Alternative Approaches

Although Python doesn't support method overloading, you can achieve similar functionality by using default arguments or variable-length arguments.

Using Default Arguments


class Test:
    def m1(self, x=None, y=None):
        if x is not None and y is not None:
            print('Two-argument method')
        elif x is not None:
            print('One-argument method')
        else:
            print('No-argument method')

t = Test()
t.m1()            # Output: No-argument method
t.m1(10)          # Output: One-argument method
t.m1(10, 20)      # Output: Two-argument method

Using Variable-Length Arguments


class Test:
    def m1(self, *args):
        if len(args) == 0:
            print('No-argument method')
        elif len(args) == 1:
            print('One-argument method')
        elif len(args) == 2:
            print('Two-argument method')
        else:
            print('Method with {} arguments'.format(len(args)))

t = Test()
t.m1()                  # Output: No-argument method
t.m1(10)                # Output: One-argument method
t.m1(10, 20)            # Output: Two-argument method
t.m1(10, 20, 30)        # Output: Method with 3 arguments

Constructor Overloading in Python

Similar to method overloading, Python does not support constructor overloading. If you define multiple constructors (i.e., multiple __init__ methods) in a class, the last one defined will overwrite the previous ones.

Example


class Test:
    def __init__(self):
        print('No-argument constructor')

    def __init__(self, x):
        print('One-argument constructor')

    def __init__(self, x, y):
        print('Two-argument constructor')

# t = Test()        # TypeError: __init__() missing 2 required positional arguments: 'x' and 'y'
# t = Test(10)      # TypeError: __init__() missing 1 required positional argument: 'y'
t = Test(10, 20)    # Output: Two-argument constructor

In the above code, only the last __init__ method is considered, and the previous ones are ignored.

Alternative Approaches

You can use default arguments or variable-length arguments in the constructor to simulate constructor overloading.

Using Default Arguments


class Test:
    def __init__(self, x=None, y=None):
        if x is not None and y is not None:
            print('Two-argument constructor')
        elif x is not None:
            print('One-argument constructor')
        else:
            print('No-argument constructor')

t1 = Test()           # Output: No-argument constructor
t2 = Test(10)         # Output: One-argument constructor
t3 = Test(10, 20)     # Output: Two-argument constructor

Using Variable-Length Arguments


class Test:
    def __init__(self, *args):
        if len(args) == 0:
            print('No-argument constructor')
        elif len(args) == 1:
            print('One-argument constructor')
        elif len(args) == 2:
            print('Two-argument constructor')
        else:
            print('Constructor with {} arguments'.format(len(args)))

t1 = Test()                  # Output: No-argument constructor
t2 = Test(10)                # Output: One-argument constructor
t3 = Test(10, 20)            # Output: Two-argument constructor
t4 = Test(10, 20, 30)        # Output: Constructor with 3 arguments

Method Overriding

Method overriding allows a child class to provide a specific implementation of a method that is already defined in its parent class. The child's method overrides the parent's method.

Example


class Parent:
    def p roperty(self):
        print('Cash + Gold + Land')

    def marry(self):
        print('Subbalakshmi')

class Child(Parent):
    def marry(self):
        print('Katrina Kaif')

c = Child()
c.property()  # Output: Cash + Gold + Land
c.marry()     # Output: Katrina Kaif

In this example, the Child class overrides the marry method of the Parent class.

Constructor Overriding

Similar to methods, constructors can also be overridden. If a child class defines its own constructor, it will override the constructor of the parent class.

Example


class Parent:
    def __init__(self):
        print('Parent Constructor')

class Child(Parent):
    def __init__(self):
        print('Child Constructor')

c = Child()  # Output: Child Constructor

In this example, the Child class constructor overrides the Parent class constructor.

Accessing Parent Class Methods

You can call the parent class method from the child class using the super() function.


class Parent:
    def property(self):
        print('Cash + Gold + Land')

    def marry(self):
        print('Subbalakshmi')

class Child(Parent):
    def marry(self):
        super().marry()
        print('Katrina Kaif')

c = Child()
c.marry()

Output:


Subbalakshmi
Katrina Kaif

In this example, the Child class's marry method calls the parent's marry method using super() before executing its own implementation.

Conclusion

Polymorphism is a fundamental concept in object-oriented programming that allows objects and methods to take on multiple forms. In Python, polymorphism is achieved through operator overloading, method overriding, and by utilizing the dynamic nature of the language.

While Python does not support method and constructor overloading in the traditional sense, you can achieve similar functionality using default arguments and variable-length arguments.

Understanding polymorphism in Python enables you to write more flexible and reusable code, leveraging the language's capabilities to handle different data types and class hierarchies effectively.
